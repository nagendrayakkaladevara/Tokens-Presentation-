What is a Token?

(Before 5 Years)

What is a Token?

A token is the smallest meaningful text unit an LLM processes.

The model does NOT read full sentences directly.

Sentence
→
Broken into Tokens
→
Processed by AI

Example

Sentence: I love Angular

Possible tokens:

I love Angular

Notice:

Tokens may contain spaces

This is VERY important.

Tokens Are NOT Always Words

Example

unbelievable

may become:

un believ able

Tokens are not always letters too.

Code Tokenization Example

TypeScript 
const user = fetchData();

Possible tokens:

Token
  • const
  • user
  • =
  • fetch
  • Data
  • ()
  • ;

Important Insight

Tokenizer is NOT grammar-based.

It is pattern/statistics-based.

It learns:

common words common subwords frequent patterns code syntax punctuation

Common Tokenization Algorithms

Modern LLMs use algorithms like:

Algorithm Used By
BPE (Byte Pair Encoding) GPT
SentencePiece Gemini, T5

Why Not Character-by-Character?

Character-level tokens:

A n g u l a r

would create:

huge sequences slow processing poor semantic learning

So subword tokenization is smarter.

Why Spaces Become Tokens

This confuses many developers initially.

Important Fact

In GPT-style tokenizers:

"cat" " cat"

are DIFFERENT tokens.

Why This Design Exists

Because spaces carry information.

Example — New York

may learn

New York
vs

instead of

New Y o r k

Another Example

Different statistically:

apple Apple

One may appear:

mid-sentence sentence start title case

Spacing + casing patterns matter in language.

Token IDs

Tokens are still text.

AI models cannot compute directly on strings.

So every token maps to an integer value called a Token ID.

Vocabulary

The model maintains a giant lookup table called a Vocabulary.

Token ID
Angular 8821
is 120
awesome 5012

Example

Input sentence: Angular is awesome

After tokenization:

["Angular", " is", " awesome"]

After converting to Token IDs:

[8821, 120, 5012]

Why Token IDs Are Needed

Neural networks can only process:

numbers vectors matrices

They cannot directly understand strings like:

"Angular"

So text must first become integers before mathematical computation begins.

Real Example

Sentence: I love Angular

Possible tokens:

["I", " love", " Angular"]

Possible Token IDs:

[40, 2045, 8821]

These IDs are then passed into the embedding layer.

Embeddings

Now we reach the MOST IMPORTANT concept in modern AI systems.

Problem

Token IDs are just meaningless integers.

TokenToken ID
Angular8821
React9231
Dog44

Example

8821 and 9231

Mathematically, these are just random integers.

The model cannot understand that:

Angular and React are related

because:

8821 ≠ semantically close to 9231

There is no meaningful relationship between token IDs themselves.

Why This Is a Problem

Neural networks work using mathematics:

distances vector operations matrix multiplication similarity calculations

But integer IDs alone cannot represent meaning.

Example

Dog → 44

Cat → 51

The numbers 44 and 51 being numerically close is completely accidental.

It does NOT mean Dog is semantically similar to Cat.

Solution: Embeddings

We convert token IDs into dense numerical vectors called Embeddings.

What is an Embedding?

A high-dimensional vector representing semantic meaning.

Angular

↓

[0.12, -0.91, 0.44, 1.82, ...]

Instead of a single integer:

8821

the token becomes a vector of many floating-point numbers.

Embedding Conversion Flow

Token
→
Token ID
→
Embedding Layer
→
Dense Vector
TokenToken IDEmbedding
Angular8821[0.21, -0.88, 1.02, ...]
React9231[0.19, -0.91, 0.97, ...]

Notice:

The vectors become similar.

Why Similar Vectors Matter

During training, the model learns:

Angular and React appear in similar contexts

Example training data:

Angular is a frontend framework

React is a frontend library

Because they appear in related contexts repeatedly, their embeddings gradually move closer together in vector space.

Embeddings Capture Semantic Meaning

semantic relationships contextual similarity syntax patterns language structure conceptual relationships

Example of Semantic Similarity

WordSimilar Words
AngularReact, Vue
DogCat, Puppy
CarVehicle, Truck

Their embedding vectors become mathematically close.

Embedding Dimensions

Real embeddings are large vectors.

ModelEmbedding Size

[0.12, -0.44, 1.87, 0.91, ...]

may contain thousands of dimensions.

Important Insight

Embeddings are NOT manually programmed.

The model learns them automatically during training.

As training progresses:

similar concepts ↓ move closer together

inside vector space.

Famous Embedding Relationship

A classic example:

King - Man + Woman ≈ Queen

Embeddings learn relational patterns mathematically.

Key Understanding

Token IDs are just indexes.

Embeddings are where semantic meaning begins.

Without embeddings:

AI cannot understand relationships between words

Embeddings transform language into mathematical representations that neural networks can process efficiently.
